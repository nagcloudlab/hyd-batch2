✅ Chapter 3 Projects Overview — Custom Authentication

🔹 chapter03.00-calendar

  • Base Project for Chapter 3.
  • Migrated to Spring Boot.
  • UserContext is still using UserContextStub.
  • No actual authentication yet (acts as a reference start state).

🔹 chapter03.01-calendar

  • Introduces Spring Security integration.
  • Adds spring-boot-starter-security dependency.
  • Adds basic authentication with form login.
  • All URLs are secured (anyRequest().authenticated()).
  • UserDetailsManager is used for in-memory authentication.
  • Still using UserContextStub.

🔹 chapter03.02-calendar

  • Demonstrates programmatic login using SecurityContextHolder.
  • When a user signs up, they are:
    1. Saved in CalendarUserDao
    1. Registered with Spring Security using UserDetailsManager
    1. Logged in programmatically via UsernamePasswordAuthenticationToken
  • Introduces real login flow after registration.

🔹 chapter03.03-calendar

  • Adds real user lookup by implementing a custom UserDetailsService.
  • Introduces CalendarUserDetailsService that queries user from DB.
  • Replaces static UserDetailsManager logic.
  • Also adds CalendarUserAuthorityUtils for dynamic role assignment.
  • Uses ROLE_ADMIN if the email starts with "admin".

🔹 chapter03.04-calendar

  • Adds a custom AuthenticationProvider:
    • Implements AuthenticationProvider.
    • Replaces DaoAuthenticationProvider.
    • Directly authenticates CalendarUser using app's service and plain-text password.
  • Can handle more complex authentication logic if needed.

🔹 chapter03.05-calendar

  • Introduces a custom authentication token:
    • Implements DomainUsernamePasswordAuthenticationToken.
    • Adds support for multi-tenant authentication (e.g., email + domain).
  • Custom provider uses both email and domain for authentication.

🔹 chapter03.06-calendar

  • Final polished version of Chapter 3:
    • Uses:
      • CalendarUserAuthenticationProvider
      • CalendarUserDetailsService
      • Custom login form
    • Real UserContext wired to SpringSecurityUserContext.
    • Fully functioning:
      • Custom authentication
      • Real-time user context mapping
      • Login + registration flow using Spring Security
      • Minimal external dependencies
    • Login roles and identity are enforced end-to-end.
