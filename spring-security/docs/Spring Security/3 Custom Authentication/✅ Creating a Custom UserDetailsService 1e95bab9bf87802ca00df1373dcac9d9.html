✅ Creating a Custom UserDetailsService

🧭 Why this matters:

Spring Security's built-in user management (like InMemoryUserDetailsManager) is great for demos, but real applications need to authenticate users from a database.

By implementing a custom UserDetailsService, you bridge your app's user model (CalendarUser) with Spring Security.

🔍 What Is UserDetailsService?

It’s a Spring Security interface that retrieves user details from a backend (e.g., DB, LDAP).



public interface UserDetailsService {
    UserDetails loadUserByUsername(String username) throws UsernameNotFoundException;
}
  • Spring calls this when it needs to authenticate a user (e.g., during login).
  • You provide the logic to:
    • Fetch your app's user (CalendarUser)
    • Convert it into Spring’s UserDetails

✅ Why Not Just Use UserDetailsManager?

FeatureUserDetailsServiceUserDetailsManager
Read-only user lookup✅ Yes✅ Yes
Add/delete users❌ No✅ Yes
Login-time use only✅❌ Used for full user management
For authentication, Spring primarily uses UserDetailsService during login.

🔧 Create CalendarUserDetailsService

📄 com.example.calendarapp.service.CalendarUserDetailsService



@Component
public class CalendarUserDetailsService implements UserDetailsService {

    private final CalendarUserDao calendarUserDao;

    public CalendarUserDetailsService(CalendarUserDao calendarUserDao) {
        if (calendarUserDao == null) {
            throw new IllegalArgumentException("calendarUserDao cannot be null");
        }
        this.calendarUserDao = calendarUserDao;
    }

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        CalendarUser user = calendarUserDao.findUserByEmail(username);
        if (user == null) {
            throw new UsernameNotFoundException("Invalid username/password.");
        }

        Collection<? extends GrantedAuthority> authorities =
            CalendarUserAuthorityUtils.createAuthorities(user);

        return new User(user.getEmail(), user.getPassword(), authorities);
    }
}

🧠 Key Logic Breakdown

StepPurpose
calendarUserDao.findUserByEmail(...)Fetch domain user from database
CalendarUserAuthorityUtils.createAuthorities(...)Create roles dynamically
new User(...)Return Spring Security-compatible object

🔧 Role Mapping via CalendarUserAuthorityUtils

Simple logic to demonstrate role assignment:



public static List<GrantedAuthority> createAuthorities(CalendarUser user) {
    if (user.getEmail().startsWith("admin")) {
        return AuthorityUtils.createAuthorityList("ROLE_ADMIN", "ROLE_USER");
    }
    return AuthorityUtils.createAuthorityList("ROLE_USER");
}
📌 You can later enhance this to pull roles from a table, LDAP group, or a REST service.

⚙️ Spring Security Integration

Spring will auto-detect this bean due to @Component. But to ensure it's used:



@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Autowired
    private CalendarUserDetailsService userDetailsService;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authz -> authz.anyRequest().authenticated())
            .formLogin();
        return http.build();
    }

    @Bean
    public AuthenticationManager authManager(HttpSecurity http) throws Exception {
        AuthenticationManagerBuilder builder = http.getSharedObject(AuthenticationManagerBuilder.class);
        builder.userDetailsService(userDetailsService)
               .passwordEncoder(passwordEncoder());
        return builder.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(); // Always hash passwords!
    }
}

🔥 SecurityConfig Highlights

  • Registers CalendarUserDetailsService as the authentication source.
  • Uses BCryptPasswordEncoder (recommended for password hashing).
  • Enables form login.
