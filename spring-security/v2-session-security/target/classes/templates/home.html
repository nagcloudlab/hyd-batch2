V2 - Session-Based Security Demo

Improvement over V1: Private endpoints now require login! Spring Security uses JSESSIONID cookie for session tracking.
How it works:
1. User submits username/password via form login
2. Server creates a session and sends JSESSIONID cookie
3. Browser sends cookie with every request - server knows who you are
4. Logout invalidates the session and deletes the cookie
