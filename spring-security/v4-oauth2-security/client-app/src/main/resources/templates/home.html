OAuth2 Authorization Code Flow:

  1. User clicks "Login with OAuth2" on Client App (:8080)
  2. Client App redirects to Auth Server (:9000) login page
  3. User enters username/password at Auth Server
  4. Auth Server shows consent screen (approve scopes)
  5. Auth Server redirects back with authorization code
  6. Client App exchanges code for access token (server-to-server)
  7. Client App calls Resource Server (:8081) with Authorization: Bearer <token>

V2 vs V3 vs V4 Comparison:

V2 - Session
- App stores passwords
- JSESSIONID cookie
- Single app only
V3 - JWT
- App stores passwords
- Self-signed JWT
- Stateless, but still manages users
V4 - OAuth2
- Auth Server manages passwords
- Signed by Auth Server
- Multiple services, SSO

cURL - Client Credentials Flow (service-to-service):

# Get token using client credentials (no user involved)
curl -X POST http://localhost:9000/oauth2/token \
  -u "service-client:service-secret" \
  -d "grant_type=client_credentials&scope=read"

# Use token to call Resource Server
curl http://localhost:8081/api/private/orders \
  -H "Authorization: Bearer <TOKEN>"

Start all 3 services:

# Terminal 1 - Auth Server
cd auth-server && mvn spring-boot:run

# Terminal 2 - Resource Server
cd resource-server && mvn spring-boot:run

# Terminal 3 - Client App
cd client-app && mvn spring-boot:run
