V3 - JWT Token Security Demo

What changed from V2 (Session)? No more JSESSIONID cookie! Now the client gets a JWT token and sends it in the Authorization: Bearer <token> header. The server is stateless - it does not store any session.
V2 Session-Based:
- Server stores session in memory
- Cookie: JSESSIONID=abc123
- Server must look up session on each request
- Hard to scale (sticky sessions needed)
V3 JWT-Based:
- Server stores NOTHING (stateless)
- Header: Authorization: Bearer eyJhb...
- Token contains all user info (self-contained)
- Easy to scale (any server can verify)
