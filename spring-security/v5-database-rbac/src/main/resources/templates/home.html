V5 - Database RBAC Demo

What changed from V2? Instead of in-memory users, we now store Users and Roles in an H2 database with JPA. New users can register, and admins can manage users/roles dynamically!
V2 - In-Memory Users
- Users hardcoded in SecurityConfig
- 2 roles: USER, ADMIN
- No registration
- Restart = reset users
V5 - Database RBAC
- Users in H2 database (JPA)
- 3 roles: USER, MANAGER, ADMIN
- Self-registration page
- Admin panel to manage users/roles
- Enable/disable users
- Custom UserDetailsService

3 Roles with Hierarchy:

RoleCan Access
USER Dashboard, Profile
MANAGER Dashboard, Profile, Manager Page
ADMIN Dashboard, Profile, Manager Page, Admin Panel (user management)
