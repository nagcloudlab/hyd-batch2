project : transfer-service-v6 (Spring Web — Thymeleaf MVC)
-------------------------------------------------------------

Reference: https://nagcloudlab.notion.site/hyd-batch-2-34c5bab9bf87804bae9ac63ad12f2a7d


Project Structure
-----------------

1. Data Model (JPA entities)
        - Account (@Entity, @Id)
        - TxnHistory (@Entity, @ManyToOne, @Enumerated)
        - TransferType (enum: DEBIT, CREDIT)

2. Layers
        1. Repository Layer   => Spring Data JPA
                - AccountRepository (JpaRepository<Account, String>)
                - TxnHistoryRepository (JpaRepository<TxnHistory, Long>)
        2. Service Layer       => business logic with @Transactional
                - TransferService (interface)
                - TransferServiceImpl (@Service)
        3. Web Layer           => Spring MVC + Thymeleaf
                - TransferController (@Controller)
                - TransferRequest (DTO with Bean Validation)
                - GlobalExceptionHandler (@ControllerAdvice)
        4. View Layer          => Thymeleaf templates + static HTML
                - templates/transfer-form.html (th:object, th:field, th:errors)
                - templates/transfer-success.html (flash attributes)
                - templates/account-list.html (th:each)
                - templates/account-detail.html (@PathVariable)
                - templates/transfer-history.html (th:each, th:classappend)
                - templates/error.html (@ExceptionHandler target)
                - static/index.html (plain HTML, no Thymeleaf)


How to Run
----------

1. Start Postgres (see v3 Notes.txt for Docker setup)
2. Run: mvn spring-boot:run
3. Open: http://localhost:8080


--------------------------------------------------------------
Spring Web Concepts Covered
--------------------------------------------------------------

1.  @Controller              — returns view names (vs @RestController for JSON)
2.  @GetMapping              — shorthand for @RequestMapping(method=GET)
3.  @PostMapping             — shorthand for @RequestMapping(method=POST)
4.  Model                    — pass data from controller to Thymeleaf template
5.  @ModelAttribute          — bind form fields to a Java object (TransferRequest)
6.  @RequestParam            — bind individual query/form params (shown in original code)
7.  @PathVariable            — extract values from URL path (/accounts/{id})
8.  @Valid + BindingResult   — Bean Validation with error capture
9.  PRG Pattern              — redirect: after POST to prevent duplicate submission
10. RedirectAttributes       — flash attributes that survive a redirect
11. @ControllerAdvice        — centralized exception handling across all controllers
12. @ExceptionHandler        — handle specific exception types, return error views

--------------------------------------------------------------
Thymeleaf Concepts Covered
--------------------------------------------------------------

1.  th:text       — safely render text (auto-escapes HTML to prevent XSS)
2.  th:if         — conditional rendering
3.  th:each       — iterate over collections
4.  th:object     — bind form to a model attribute
5.  th:field      — bind input to a field (generates name/id/value)
6.  th:errors     — show validation error messages
7.  th:action     — generate form action URL
8.  th:href       — generate dynamic links with @{...} URL syntax
9.  th:classappend — conditionally add CSS classes
10. |...|         — literal substitution (string interpolation)
11. ${...}        — variable expression (access model attributes)
12. *{...}        — selection expression (access fields on th:object)
13. @{...}        — URL expression (context-path aware)


--------------------------------------------------------------
Request Flow
--------------------------------------------------------------

Browser -> Embedded Tomcat -> DispatcherServlet -> @Controller method
   -> return "view-name"
   -> ViewResolver (Thymeleaf)
   -> templates/view-name.html (processed with Model data)
   -> HTML response -> Browser
